# Documentation Files Convention

**IMPORTANT**: All documentation/summary markdown files (*.md) MUST be created in the `/documentation` folder at the root of the project.

## Rules:
1. Never create .md files in the root directory (except README.md)
2. All guides, summaries, explanations, and documentation go in `/documentation`
3. Only create README.md in the root if it's the main project README

## Exceptions:
- `README.md` (root) - Main project README only
- `backend/README.md` - Backend-specific README

## Examples:
✅ CORRECT:
- `/documentation/APP-ANALYSIS.md`
- `/documentation/MOBILE-APP-GUIDE.md`
- `/documentation/START-HERE.md`

❌ WRONG:
- `/APP-ANALYSIS.md`
- `/SETUP-COMPLETE.md`
- `/GUIDE.md`
